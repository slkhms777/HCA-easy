## 2. MANO 参数（可选，可忽略）

路径：`datasets/calibration/mano`

每个被试的 `betas` 文件位于：

```text
datasets/calibration/mano/{subject_name}
```

如果该文件不存在，则默认使用平均手型参数。

示例格式：

```yaml
betas:
  - -0.5082021951675415
  - -0.24375076591968536
  - -0.31839972734451294
  - -0.17650160193443298
  - 0.4463734030723572
  - 0.05645660683512688
  - -0.6338333487510681
  - 0.15932975709438324
  - -0.18557298183441162
  - -0.08094348013401031
```

## 3. 物体 CAD 模型

路径：`datasets/models`

目录结构示例：

```text
datasets/models/G01_1/
├── textured_mesh.mtl
├── textured_mesh.obj
└── *.png 或 *.jpg
```

## 4. 图片序列数据

目录结构示例：

```text
datasets/{subject_name}/{data_name}/
├── serial_1/
│   ├── color_{frame_idx:06d}.jpg
│   └── depth_{frame_idx:06d}.png
├── serial_2/
├── ...
├── serial_8/
├── processed/      # 处理中间结果
└── initial_poses/  # 最终导出的初始手-物姿态
```

`color` 和 `depth` 需要是相同的 size，并且已经提前时序对齐过。

## 5. 元数据

路径：`datasets/{subject_name}/{data_name}/meta.yaml`

示例格式：

```yaml
realsense:
  serials:
    - '037522251142'
    - '043422252387'
    - '046122250168'
    - '105322251225'
    - '105322251564'
    - '108222250342'
    - '115422250549'
    - '117222250549'
  width: 640
  height: 480
extrinsics: datasets/calibration/extrinsics/subject_5
subject_id: subject_5
object_ids:
  - G15_1
  - G15_2
  - G15_3
  - G15_4
mano_sides:
  - right
# 双手示例：
# mano_sides:
#   - right
#   - left
num_frames: 702
task_id: 1
```

说明：

- `object_ids` 表示场景中参与交互的物体 CAD 模型 ID。
- `mano_sides` 用于指定当前序列中需要拟合的手，支持单手或双手。
- `task_id` 任务类型标记，可随便填。
